================================================================================
  front 폴더 — HTML 파일 이름 ↔ 화면(기능) 정리
================================================================================

※ 기본면접 / 실전면접 종료 후 「🎉 수고하셨어요!」와 결과·피드백 UI는 각각
   별도 HTML로 분리되어 있습니다.

  · 기본면접(영상 면접) 종료 후 결과 페이지 → normal-result.html
  · 실전면접(다면접관 화면) 종료 후 결과 페이지 → basic-result.html

※ 코드·파일명 쪽에서는 예전부터 이렇게 붙어 있어 혼동하기 쉽습니다.

  · 화면에서 「기본면접」카드 → 진행: video-interview.html  → 결과: normal-result.html
  · 화면에서 「실전면접」카드 → 진행: basic-interview.html   → 결과: basic-result.html

  (파일 이름의 video / normal 은 기본면접 플로우, basic 은 실전면접 플로우에 대응)

--------------------------------------------------------------------------------
파일명                    | 사용자 입장에서의 화면
--------------------------------------------------------------------------------
dashboard.html            | 랜딩(홈), 요약 카드, 면접 태도·역량·피드백 모달, 로그인 모달
mock-interview.html       | 모의면접 — 빠른/기본/실전 면접 유형 선택
quick-interview.html      | 빠른면접 진행
quick-result.html         | 빠른면접 종료 후 결과(수고하셨어요 + 빠른면접 피드백 UI)
video-interview.html      | 기본면접 진행(면접관·나 화면, vi-* 스타일)
normal-result.html        | 기본면접 종료 후 결과(수고하셨어요 + 전달력/내용 등, ns-* 데이터)
basic-interview.html      | 실전면접 진행(AI 면접관 3인 등, bi-* 스타일)
basic-result.html         | 실전면접 종료 후 결과(수고하셨어요 + 레이더·AI 피드백 블록 등)
interview-analysis.html   | 면접분석(평균 점수, 태도/역량 상세 레이아웃)
ai-feedback.html          | AI 피드백 전용 페이지(날짜 탭·강점·보완점·추천 학습)
interview-record.html     | 면접기록 목록·통계
mypage.html               | 마이페이지(이름·이메일, GET /api/auth/me 동기화), 상단 메뉴에서 이동
signup.html               | 회원가입
api.js                    | API_BASE, localStorage 토큰·유저, Bearer 붙는 apiAuthFetch 및 면접 API 래퍼
--------------------------------------------------------------------------------
