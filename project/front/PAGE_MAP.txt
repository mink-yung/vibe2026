================================================================================
  front 폴더 — 페이지·스크립트·백엔드 API 연동 정리 (현재 구현 기준)
================================================================================

※ 기본면접 / 실전면접 종료 후 「🎉 수고하셨어요!」와 결과·피드백 UI는 각각
   별도 HTML로 분리되어 있습니다.

  · 기본면접(영상 면접) 종료 후 결과 → normal-result.html
  · 실전면접(다면접관 화면) 종료 후 결과 → basic-result.html

※ 코드·파일명과 화면 라벨이 다릅니다 (혼동 주의).

  · 화면 「기본면접」 → 진행: video-interview.html  → 결과: normal-result.html
  · 화면 「실전면접」 → 진행: basic-interview.html   → 결과: basic-result.html
  · 화면 「빠른면접」 → 진행: quick-interview.html  → 결과: quick-result.html

  (파일명 video/normal = 기본면접 플로우, basic = 실전면접 플로우)

--------------------------------------------------------------------------------
공통 UI
--------------------------------------------------------------------------------
  · 좌측 사이드바: 홈 / 모의면접 / 면접분석 / AI피드백 / 면접기록 (대부분 HTML 공통)
  · 상단 authTopbar: dashboard.js — 로그인 시 이름·마이페이지 링크, 비로그인 시 로그인 버튼
  · 로그인 모달: dashboard.html 등 — POST /api/auth/login (Bearer 없이 fetch)
  · 인증 저장: api.js — localStorage `token`, `user` (JWT + 사용자 정보)

--------------------------------------------------------------------------------
면접 플로우 (로그인 필요 — 면접 API는 토큰 필수)
--------------------------------------------------------------------------------

  mock-interview.html
       ├─ 빠른면접  → quick-interview.html  ─[종료]→ POST /interviews/quick  → quick-result.html
       ├─ 기본면접  → video-interview.html  ─[종료]→ POST /interviews/basic → normal-result.html
       └─ 실전면접  → basic-interview.html  ─[종료]→ POST /interviews/real   → basic-result.html

  · 진행 중 「나가기」(video / basic): 경고 모달 — 저장 안 됨 안내, 「다시 보지 않기」 빨간 버튼
    (localStorage: interviewExitWarnDontShow_basic | _real)
  · 영상·음성 STT: UI만 있음. 실제 제출은 화면 텍스트/플레이스홀더 답변으로 API 호출

공통·전용 스크립트
--------------------------------------------------------------------------------
api.js
  · API_BASE = https://squad-understanding-peaceful-power.trycloudflare.com/
  · getStoredAuthToken / saveAuthToStorage / clearStoredAuth
  · apiAuthFetch — 인증 API (Bearer 선택)
  · apiInterviewFetch — 면접 API (Bearer 필수)
  · 래퍼: auth/me, profile GET·PATCH, account DELETE,
          interviews quick·basic·real POST, history·recent/analysis·summary GET

dashboard.js
  · showSection() — 현재 페이지에 섹션 없으면 SECTION_PAGE 경로로 이동
  · 로그인 모달·탑바·차트·모달(태도/역량/피드백)
  · finishBasicVideoInterviewSubmit / finishRealInterviewSubmit
  · 면접 나가기 모달 (openInterviewExitWarnModal)
  · initDashboardInterviewApi — 홈 카드·최근 기록
  · initInterviewAnalysisStandaloneApi — interview-analysis.html
  · initAiFeedbackInlinePage — ai-feedback.html (data-page="ai-feedback")

quick-interview.js    · finishQuickInterviewSubmit → quick-result 이동
quick-result.js       · sessionStorage `quickInterviewLastResult` 렌더
normal-result.js      · sessionStorage `normalInterviewLastResult` 렌더
basic-result.js       · sessionStorage `basicInterviewLastResult` 렌더 (3인 피드백 매핑)
interview-record.js   · history → 통계·테이블 (quick/basic/real → 유형 뱃지)
mypage.js             · 계정정보·상세정보(조회/수정/취소 모달)·회원탈퇴 모달

--------------------------------------------------------------------------------
마이페이지 (mypage.html) 탭
--------------------------------------------------------------------------------
  계정정보   · GET /api/auth/me — 이름, 이메일, 가입일
  상세정보   · GET /api/auth/profile — 조회·요약 보기 / 수정·PATCH / 작성 취소 모달
             · 필드: 나이, 출신대학, 전공, 취미, 특기, 희망직무, 경력수준, 기술/역량,
               자격증, 프로젝트, 경력/활동, 이력서, 포트폴리오·깃허브 URL
  회원탈퇴   · DELETE /api/auth/account — 확인 모달 후 삭제 → 홈 이동

프론트에서 아직 미연동 (백엔드 api.txt 계획 항목)
--------------------------------------------------------------------------------
  · GET    /api/interviews/:interviewId  — 면접 상세
  · DELETE /api/interviews/:interviewId  — 면접 기록 삭제
  · POST   /api/interviews/quick/audio   — 빠른면접 음성
  · POST   /api/interviews/basic/audio   — 기본면접 음성
  · GET    /api/personas                 — 페르소나 목록 (백엔드만 존재)
